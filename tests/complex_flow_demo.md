# 用户登录与权限校验流程

用户输入账号和密码。
系统检查登录信息是否完整。
如果登录信息不完整，则提示用户重新输入。
如果登录信息完整，则调用登录验证模块。

系统判断账号密码是否正确。
如果账号密码错误，则提示用户重新登录。
如果账号密码正确，则判断用户是否具有访问权限。

如果用户没有访问权限，则显示无权限提示。
如果用户具有访问权限，则进入系统首页并加载用户菜单。

# 文件上传与解析流程

用户上传文件。
系统检查文件是否为空。
如果文件为空，则提示用户重新上传。
如果文件不为空，则检查文件格式是否支持。

如果文件格式不支持，则提示用户重新上传。
如果文件格式支持，则检查文件大小是否超过限制。

如果文件大小超过限制，则提示用户压缩文件后重新上传。
如果文件大小没有超过限制，则调用文件解析模块。

系统判断文件解析是否成功。
如果解析失败，则调用错误记录模块并提示用户重新上传。
如果解析成功，则输出解析结果并保存文件信息。

# 项目审批流程

用户提交项目申请。
系统检查申请材料是否完整。
如果申请材料不完整，则提示用户补充材料。
如果申请材料完整，则提交给导师审核。

导师判断项目是否符合要求。
如果项目不符合要求，则退回给用户修改。
如果项目符合要求，则提交给学院审核。

学院判断项目是否需要进一步评审。
如果需要进一步评审，则安排专家评审。
如果不需要进一步评审，则直接通过审批。

专家判断项目是否通过评审。
如果评审不通过，则退回给用户修改。
如果评审通过，则生成审批结果并通知用户。

# Agent 流程图生成流程

用户输入流程描述或上传文档。
系统判断输入内容是否为空。
如果输入为空，则提示用户重新输入。
如果输入不为空，则调用 Flow Segmenter 切分流程。

系统判断是否检测到多个流程。
如果检测到多个流程，则逐个处理每个流程片段。
如果只检测到一个流程，则直接处理默认流程。

系统调用 Research Agent 抽取关键概念。
系统判断 concepts 是否为空。
如果 concepts 为空，则触发 retry 重新抽取。
如果 concepts 不为空，则调用 Decomposition Agent 进行系统拆解。

系统调用 Router 判断流程类型。
如果流程类型是 linear，则调用 linear rule extractor。
如果流程类型是 branch，则调用 branch flow extractor。
如果流程类型不支持，则提示暂不支持该流程类型。

系统生成 Mermaid 代码。
系统检查 Mermaid 是否可以渲染。
如果不能渲染，则提示用户检查流程描述或 Mermaid 代码。
如果可以渲染，则保存 Mermaid 文件和 SVG 图片。
