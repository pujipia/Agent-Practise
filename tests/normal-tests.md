# TEST 4：复杂文件上传与解析流程 (多重判斷)

用户上传文件。
系统判断文件是否为空。

如果文件为空，则系统提示用户重新上传。
提示完成后，流程返回到“用户上传文件”。

如果文件不为空，则系统判断文件格式是否支持。

如果文件格式不支持，则系统提示用户重新上传。
提示完成后，流程返回到“用户上传文件”。

如果文件格式支持，则系统判断文件大小是否超过限制。

如果文件大小超过限制，则系统提示用户压缩文件后重新上传。
提示完成后，流程返回到“用户上传文件”。

如果文件大小未超过限制，则系统调用文件解析模块。
文件解析完成后，系统判断文件解析是否成功。

如果文件解析失败，则系统调用错误记录模块并提示用户重新上传。
提示完成后，流程返回到“用户上传文件”。

如果文件解析成功，则系统输出解析结果并保存文件信息。
输出完成后，流程结束。

# TEST 5：登录验证与验证码流程 (混合邏輯)

用户进入登录页面。
用户输入账号和密码。
系统判断账号或密码是否为空。

如果账号或密码为空，则系统提示用户重新输入账号和密码。
提示完成后，流程返回到“用户输入账号和密码”。

如果账号或密码不为空，则系统调用登录验证模块。
登录验证模块完成后，系统判断账号密码是否正确。

如果账号密码错误，则系统记录一次登录失败次数。
记录失败次数完成后，系统判断失败次数是否超过限制。

如果失败次数超过限制，则系统锁定账号并提示用户联系客服。
账号锁定后，流程结束。

如果失败次数未超过限制，则系统提示登录失败。
提示完成后，流程返回到“用户输入账号和密码”。

如果账号密码正确，则系统判断是否需要验证码。

如果需要验证码，则系统发送验证码并提示用户输入验证码。
用户输入验证码后，系统判断验证码是否正确。

如果验证码错误，则系统提示验证码错误。
提示完成后，流程返回到“用户输入验证码”。

如果验证码正确，则系统进入系统首页。
进入系统首页后，流程结束。

如果不需要验证码，则系统直接进入系统首页。
进入系统首页后，流程结束。

# TEST 6：订单库存与支付流程

用户进入订单确认页面。
用户提交订单。
系统判断订单信息是否完整。

如果订单信息不完整，则系统提示用户补全订单信息。
提示完成后，流程返回到“用户提交订单”。

如果订单信息完整，则系统调用库存检查模块。
库存检查完成后，系统判断库存是否充足。

如果库存不足，则系统提示用户商品库存不足。
提示完成后，流程结束。

如果库存充足，则系统锁定库存。
库存锁定完成后，系统判断用户是否使用优惠券。

如果用户未使用优惠券，则系统使用原价生成支付金额。
生成支付金额后，系统调用支付模块。

如果用户使用优惠券，则系统判断优惠券是否有效。

如果优惠券无效，则系统提示用户优惠券不可用。
提示完成后，系统使用原价生成支付金额。
生成支付金额后，系统调用支付模块。

如果优惠券有效，则系统计算优惠后金额。
计算完成后，系统调用支付模块。

支付模块完成后，系统判断支付是否成功。

如果支付失败，则系统释放库存。
库存释放完成后，系统判断用户是否选择重新支付。

如果用户选择重新支付，则流程返回到“系统调用支付模块”。

如果用户不选择重新支付，则系统取消订单。
取消订单后，流程结束。

如果支付成功，则系统调用交易风控模块。
风控检查完成后，系统判断风控是否通过。

如果风控不通过，则系统冻结订单并提交人工审核。
提交人工审核后，流程结束。

如果风控通过，则系统生成订单编号。
订单编号生成后，系统通知仓库发货。
仓库处理完成后，系统判断发货是否成功。

如果发货失败，则系统创建发货异常工单。
创建异常工单后，流程结束。

如果发货成功，则系统通知用户订单已发货。
通知完成后，流程结束。

# TEST 7：合同审查 Agent 流程

用户上传合同文件。
系统判断合同文件是否成功接收。

如果合同文件未成功接收，则系统提示用户重新上传合同文件。
提示完成后，流程返回到“用户上传合同文件”。

如果合同文件成功接收，则系统判断文件格式是否支持。

如果文件格式不支持，则系统提示用户上传 PDF 或 DOCX 文件。
提示完成后，流程返回到“用户上传合同文件”。

如果文件格式支持，则系统调用合同文本解析模块。
文本解析完成后，系统判断合同文本是否为空。

如果合同文本为空，则系统提示用户检查合同文件内容。
提示完成后，流程结束。

如果合同文本不为空，则系统调用条款抽取 Agent。
条款抽取完成后，系统判断是否成功抽取关键条款。

如果未成功抽取关键条款，则系统触发条款抽取 retry。
retry 完成后，系统再次判断是否成功抽取关键条款。

如果 retry 后仍未成功抽取关键条款，则系统提示需要人工审查。
提示完成后，流程结束。

如果成功抽取关键条款，则系统调用合规风险识别 Agent。
风险识别完成后，系统判断是否发现高风险条款。

如果未发现高风险条款，则系统生成低风险审查报告。
报告生成后，流程结束。

如果发现高风险条款，则系统标注风险条款及其判断依据。
标注完成后，系统判断风险是否涉及知识产权或保密义务。

如果风险不涉及知识产权或保密义务，则系统生成普通风险提示报告。
报告生成后，流程结束。

如果风险涉及知识产权或保密义务，则系统转交法务负责人复核。
法务复核完成后，系统判断法务是否批准继续。

如果法务不批准继续，则系统生成高风险终止建议。
建议生成后，流程结束。

如果法务批准继续，则系统生成修改建议和风险提示报告。
报告生成后，流程结束。

# TEST 8：File Agent 自身处理流程

用户进入 File Agent 主菜单。
用户选择输入方式。
系统判断用户是否选择手动输入。

如果用户选择手动输入，则系统读取多行文本。
读取完成后，系统判断输入内容是否为空。

如果用户没有选择手动输入，则系统判断用户是否选择文档输入。

如果用户选择文档输入，则系统读取文件路径。
系统判断文件路径是否存在。

如果文件路径不存在，则系统提示用户重新输入文件路径。
提示完成后，流程返回到“用户选择输入方式”。

如果文件路径存在，则系统判断文件格式是否为 txt 或 md。

如果文件格式不支持，则系统提示用户上传 txt 或 md 文件。
提示完成后，流程返回到“用户选择输入方式”。

如果文件格式支持，则系统读取文档内容。
读取完成后，系统判断输入内容是否为空。

如果输入内容为空，则系统提示用户重新输入内容。
提示完成后，流程返回到“用户选择输入方式”。

如果输入内容不为空，则系统调用 Flow Segmenter。
Flow Segmenter 完成后，系统判断是否检测到多个流程。

如果检测到多个流程，则系统逐个处理每个流程片段。

如果只检测到一个流程，则系统处理默认流程。

系统调用 Research Agent 抽取关键概念。
Research Agent 完成后，系统判断 concepts 是否为空。

如果 concepts 为空，则系统触发 Research Agent retry。
retry 完成后，系统再次判断 concepts 是否为空。

如果 retry 后 concepts 仍为空，则系统提示用户流程信息不足。
提示完成后，流程结束。

如果 concepts 不为空，则系统调用 Decomposition Agent。
Decomposition Agent 完成后，系统调用 Router 判断流程类型。

系统判断流程类型是否为 linear。

如果流程类型为 linear，则系统调用 linear rule extractor。
linear rule extractor 完成后，系统生成 Mermaid 代码。

如果流程类型不是 linear，则系统判断流程类型是否为 branch。

如果流程类型为 branch，则系统调用 branch flow extractor。
branch flow extractor 完成后，系统调用 branch validator。
系统判断 branch validator 是否发现严重错误。

如果发现严重错误，则系统触发 branch retry。
branch retry 完成后，系统再次调用 branch validator。

如果 retry 后仍存在严重错误，则系统提示用户检查流程描述。
提示完成后，流程结束。

如果 retry 后没有严重错误，则系统生成 Mermaid 代码。

如果流程类型既不是 linear 也不是 branch，则系统提示暂不支持该流程类型。
提示完成后，流程结束。

Mermaid 代码生成后，系统调用 Mermaid CLI 渲染 SVG。
系统判断 SVG 是否生成成功。

如果 SVG 生成失败，则系统保存 Mermaid 文件和错误日志。
保存完成后，流程结束。

如果 SVG 生成成功，则系统保存 Mermaid 文件、SVG 图片、Research JSON 和 Decomposition JSON。
保存完成后，系统输出最终摘要。
流程结束。
